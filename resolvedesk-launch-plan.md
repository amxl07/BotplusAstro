# ResolveDesk AI Launch Plan

## 1. Current Decision

Domains acquired:

- `resolvedesk.com`
- `resolvedesk.ai`

Recommended primary domain:

> `resolvedesk.com`

Recommended secondary domain:

> `resolvedesk.ai` should redirect to `resolvedesk.com` with a permanent 301 redirect.

Reason:

- `.com` is stronger for global enterprise trust.
- `.ai` is useful defensively and for campaigns, but the canonical brand home should be `.com`.

## 2. Domain Protection Checklist

This is not legal advice, but it is the practical protection baseline.

### Registrar security

- [ ] Enable registrar account 2FA using an authenticator app or hardware security key.
- [ ] Do not use SMS-only 2FA if avoidable.
- [ ] Use a unique, long password stored in a password manager.
- [ ] Enable domain transfer lock / registrar lock.
- [ ] Enable WHOIS/domain privacy where possible.
- [ ] Turn on auto-renewal for all important domains.
- [ ] Add a backup payment method.
- [ ] Set calendar reminders 60, 30, and 7 days before renewal.
- [ ] Use an admin email that is protected by 2FA and not dependent on the domain itself.

### Advanced domain controls

- [ ] Ask registrar whether registry lock is available for `resolvedesk.com`.
- [ ] Enable DNSSEC if supported by the DNS provider and hosting stack.
- [ ] Use a reputable DNS provider with audit logs and 2FA.
- [ ] Restrict DNS access to the smallest possible number of people.
- [ ] Keep registrar, DNS, hosting, and email admin access separated if possible.

### Brand/domain defensive purchases

Buy only the highest-value defensive variants. Do not overspend too early.

High priority:

- [ ] `resolvedeskai.com` if available.
- [ ] `resolvedesk.ai` already purchased.
- [ ] Common misspellings if cheap and obvious.
- [ ] Country domains only for target markets if affordable and strategically important.

Medium priority:

- [ ] `getresolvedesk.com`
- [ ] `tryresolvedesk.com`
- [ ] `resolvedesk.io`
- [ ] `resolvedesk.co`

Low priority:

- [ ] Hyphenated versions.
- [ ] Many country-code domains with no near-term plan.
- [ ] Exotic TLDs.

### Email and anti-spoofing

Set these before sending sales emails from the domain.

- [ ] SPF.
- [ ] DKIM.
- [ ] DMARC with monitoring first, then stricter policy later.
- [ ] BIMI later if brand/email volume justifies it.
- [ ] Use separate subdomains for marketing email if needed, e.g. `mail.resolvedesk.com`.
- [ ] Do not send cold email from the primary root domain immediately at scale.

Suggested email domains/subdomains:

| Use | Domain/Subdomain |
|---|---|
| Main website | `resolvedesk.com` |
| Human business email | `name@resolvedesk.com` |
| Marketing email | `mail.resolvedesk.com` |
| Product app | `app.resolvedesk.com` |
| Help/support | `support.resolvedesk.com` or `help.resolvedesk.com` |
| Documentation | `docs.resolvedesk.com` |
| Status page | `status.resolvedesk.com` |

### Trademark and legal protection

- [ ] Run a trademark search for `ResolveDesk` and `ResolveDesk AI` in key markets.
- [ ] Check US, UK, EU, UAE, India, and any country where sales are expected soon.
- [ ] Check company registry conflicts.
- [ ] Check LinkedIn, X, YouTube, GitHub, and major social handles.
- [ ] If clear, file trademark in the most important initial jurisdiction.
- [ ] Prepare simple brand usage guidelines.

Priority jurisdictions:

1. UAE, because BotPlus is based there and early traction exists.
2. US, because it is a huge SaaS/contact centre market and trademark conflicts can be expensive.
3. UK/EU, because `contact centre` searches and enterprise buyers are strong.
4. India, because founder/operator base and possible support/BPO opportunity.

## 4. ICP Architecture

Yes, create separate ICPs by industry. Also create sub-ICPs by company size where buying motion, budget, and pain differ.

Recommended ICP structure:

```text
Industry ICP
  -> Enterprise version
  -> Mid-market version
  -> Key workflows
  -> Buying committee
  -> Trigger events
  -> Objections
  -> ROI model
  -> Pilot package
```

### Company size segmentation

| Segment | Support Profile | Buyer | Sales Motion | Best Offer |
|---|---|---|---|---|
| SMB | 1-10 agents | Owner/operator | Self-serve or light sales | Not priority now |
| Mid-market | 10-100 agents | COO, Head of CX, Ops Manager | Fast demo + pilot | 30-day workflow pilot |
| Enterprise | 100-1,000+ agents | CIO, COO, CX Director, Procurement | Consultative sales | Savings audit + phased rollout |
| Public sector | Service desks and citizen channels | Digital transformation, IT, service excellence | Relationship + tender/proposal | Citizen service AI pilot |

### Why not focus on SMB first

SMB can create noisy leads but lower contract value. ResolveDesk should lead with enterprise and mid-market because the core claim is operational cost reduction at scale.
